我系统过了一遍 dashboard 所有关键组件。按影响严重度给你列：                         
                                                          
  🔴 核心 UX 问题                                                                                                                                                             1. Header 右侧 8 个按钮过载
  Header 从左到右排了：首页 / 书签 / 导入 / 导出 / 显示设置 / 帮助 / 设置 / 关于
  header.tsx 近 500 行,单单右侧按钮就占 200 行                                        
  - "首页"和顶部 logo 点击都回首页，重复
  - "帮助"、"关于"、"设置" 是低频操作却都占一级位置
  - 在 14 英寸屏幕上按钮会挤到中部搜索框
  - 搜索框被压缩到 max-w-lg，输入"多个关键词"时就截断了

  建议：保留一级按钮 3 个（添加书签 / 导入 / 设置），其余全部塞进右侧的"更多"(⋯)      
  菜单：导出、显示、帮助、关于。动线一次减 5 个按钮。

  2. 首页"第一个子分类截 7 个书签"策略有问题

  enhanced-main-content.tsx:439-485：
  const firstSubCategory = category.subCategories[0]
  const firstSubCategoryBookmarks = ...
    .slice(0, 7)
  一个分类如果有 3 个子分类，首页只看得到 1 个子分类的 7 个书签。其他 2
  个子分类完全不曝光，用户得点 chip 或进详情页才能发现。

  - [0] 按顺序取，不是按热度/频率，体验随机
  - 7 是硬编码，不响应屏幕尺寸
  - 用户不清楚为什么别的书签"消失"了

  建议（选一）：
  - (a) 每个子分类都渲染一行 4-6 个，滚动着看所有；更符合 dashboard 概念
  - (b) 按"最近访问"或"使用频率"跨子分类混合显示 N 个；但需要 trackActivity 数据支持  
  - (c) 只显示到一级分类名 + 点击进入，不渲染书签网格；首页变成纯"目录"

  我倾向 (a)，对你这种"书签收藏夹"定位最自然。

  3. URL 不跟随导航状态 → 刷新丢失

  当前 selectedCategory / selectedSubCategory / searchQuery 全部是 React state，不写入
   URL。导致：
  - 浏览器刷新 → 回到首页所有分类视图
  - 无法分享链接 /dashboard?category=dev-tools&sub=code-editors
  - 浏览器返回/前进按钮失效

  建议：把这 3 个状态同步到 URL（useSearchParams + router.replace）。这是导航类 app   
  的基本操作。

  4. 两处"批量管理"图标完全相同但作用不同

  - Sidebar 顶部 <CheckSquare> 按钮 = 批量选分类删除
  - 主内容区分类标题旁 <CheckSquare> 按钮 = 批量选书签删除/移动/导出

  图标一样，用户容易误点。

  建议：Sidebar 那个换个更专用图标（FolderCog / FolderX），或直接挪到 Sidebar
  顶部的"更多"下拉里，因为批量删分类是罕见操作，不该占一级位置。

  5. 搜索/筛选状态栏与 Header 搜索框信息重复

  enhanced-main-content.tsx:401-430：输入 "react" 后，Header 搜索框显示
  "react"，下方又出现一条 搜索: "react" 状态栏。同一信息用了两个地方。

  建议：状态栏保留"分类路径"（面包屑功能），搜索 query
  不再重复显示，或把整个状态栏去掉由搜索结果页自己承担。

  6. 演示数据横幅过于抢眼

  🎯 这些是演示书签,帮助您了解应用功能
  您可以直接使用这些分类,或者清除后创建自己的书签收藏
  这条横幅用了 primary/5→primary/10→primary/20 三层蓝色渐变，加 emoji，占一整行。     

  建议：
  - 收缩成右下角的小气泡（DemoBadge），类似很多 B 端产品的"试用中"小标签
  - 或者更粗暴：用户加第一条自己的书签时自动隐藏，已经有 hasUserData 逻辑了，加个     
  setState 触发 rerender 即可
  - emoji "🎯" 换成 lucide <Info>

  7. Sidebar 折叠后完全失去导航能力

  // sidebar.tsx:205-215
  if (collapsed) {
    return <div className="w-12 border-r bg-muted/10">
      <Button onClick={onToggleCollapse}>
        <PanelLeft />  // 只剩一个展开按钮
      </Button>
    </div>
  }
  折叠后 w-12 的侧边栏只有一个展开按钮，毫无导航价值。用户为了快速切换分类只能先展开、
  再点、再折叠。

  建议：折叠状态展示每个一级分类的首字母或首个图标，点击直达；类似 Discord / VS Code  
  侧栏折叠时的"图标条"。

  🟡 视觉/一致性问题

  8. 胶囊子分类按钮两处样式不一致

  详情视图 (enhanced-main-content.tsx:262-278)：
  "bg-primary text-primary-foreground shadow-md hover:shadow-lg"  // 选中
  "bg-muted hover:bg-primary/10"                                   // 未选中
  首页视图 (enhanced-main-content.tsx:465-479)：
  "bg-gradient-to-r from-primary to-primary/80 ... shadow-lg"      //
  "选中"(其实永远高亮第一个)
  "hover:scale-105 hover:shadow-lg border ..."                     // 未选中
  两处视觉不一致，首页还有 hover:scale-105
  放大效果，详情页没有。用户切换视图会觉得"变了个风格"。

  建议：统一为同一组样式变量，通过 prop 控制 variant="default" | "emphasis" 之类。    

  9. 批量选择高亮没做暗色适配

  sidebar.tsx:285:
  isBatchMode && selectedCategoryIds.has(category.id) && "bg-blue-50 border
  border-blue-200"
  硬编码浅色 bg-blue-50。暗色模式下蓝色很淡几乎看不见，而且和全站 primary 色不一致。  

  建议：改 bg-primary/15 border-primary/30（语义化 + 自动适配）。

  10. 整个页面 muted 灰底层次太多

  同一页面存在的背景色：
  - bg-background（全局）
  - bg-muted/10（sidebar）
  - bg-muted/20（筛选状态栏）
  - bg-gradient-to-br from-background to-muted/20（main 背景）
  - bg-gradient-to-r from-primary/5 to-primary/10（演示横幅）

  五种淡色背景交织，视觉分层仅靠 border，深色模式下对比度更差。

  建议：去掉 main 的 gradient（直接 bg-background），只留 sidebar 一个 muted/10       
  做分区。

  11. 分类标题左边的"彩色竖条"在两个视图粗细不同

  - 详情视图 w-1.5 sm:w-2 h-5 sm:h-6
  - 首页视图 w-1 sm:w-1.5 h-4 sm:h-5

  细节处的不一致。建议：抽成 <CategoryMarker> 组件。

  12. 快捷显示按钮 (QuickDisplaySettings) 和正式设置 (SettingsPanel) 职责重叠

  Header 有 <QuickDisplaySettings>，设置面板里也有显示偏好，用户有时不知道该去哪改。  

  建议：二选一。保留 QuickDisplaySettings 放
  Header（高频），设置面板去掉重复项；或者彻底删除
  QuickDisplaySettings，全部收纳到设置面板。

  🟢 功能缺失 / 增强

  13. 没有键盘快捷键生态

  只有 Ctrl+K（搜索），其他操作全鼠标：
  - N - 新建书签
  - / - 聚焦搜索
  - Esc - 关闭对话框/退出选择模式
  - Cmd/Ctrl+A - 在选择模式下全选
  - ↑/↓ - Sidebar 分类导航

  Power user 每天用几十次的应用，应该支持。

  14. 没有面包屑

  当前导航层级只能通过 Sidebar 反推，主内容区缺"当前位置"指示。筛选状态栏
  勉强算但信息过散。

  建议：在主内容区顶部加一行真正的 Home / 开发工具 / 代码编辑器，可点击回溯。

  15. BookmarkPreview / 卡片右键菜单存在但发现性差

  onPreview 回调在卡片菜单里的 预览网站，用户得先 hover 卡片→点击三点菜单→点预览。    

  建议：卡片长按 / 右键直接弹预览，或 Shift+Click 触发。

  16. 后台增强进度条位置太隐蔽

  <EnhancementProgress> 在最底部，用户看不到"后台正在获取这些书签的封面"这个信号。    

  建议：做成 header 里搜索框旁的小 loading ring，带 count，点开展开详情。

  17. 没有 Undo

  误删一个分类 → 分类下所有书签全没了，没有恢复入口。用户得去导出文件找回。

  建议：删除时 toast 显示 5 秒 "已删除（撤销）" 按钮，简单用一个短期内存栈实现。      

  18. 移动端 sidebar 处理简陋

  w-64 固定，手机屏幕上会吃掉一半的书签区。应该：
  - < 640px 时 sidebar 默认隐藏
  - 右上一个汉堡按钮打开为 Sheet（全屏抽屉）
  - 选完分类自动关闭
