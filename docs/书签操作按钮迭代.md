## 核心问题：按钮怎么放？

**按钮越来越多 ≠ 都堆到卡片上**。把功能按"使用频率 / 是否需要批量"分三层：

### 三层放置策略

```
┌─ L1 卡片表面（hover 显示） ────────────────────────────┐
│   高频快捷, 单卡片操作, 2-3 个图标足矣              │
│   推荐：[★收藏] [✎编辑] [⧉复制]                     │
│   不放：删除（误触代价高）、新窗口打开（单击=打开）   │
└────────────────────────────────────────────────────┘

┌─ L2 卡片右键菜单 / ⋯ 备份入口 ──────────────────────┐
│   完整功能集, 低频长尾                              │
│   编辑 / 复制 / 移动 / 加标签 / 删除 / 加入收藏夹    │
└────────────────────────────────────────────────────┘

┌─ L3 多选 + 底部工具栏 ─────────────────────────────┐
│   批量操作, 一次处理多个                            │
│   现有：移动 / 导出 / 删除                          │
│   新增：批量加标签 / 批量加入收藏夹 / 批量打开       │
└────────────────────────────────────────────────────┘
```

**好处**：
- 卡片表面只 3 个高频按钮，视觉不挤
- 删除等危险操作必须经 ⋯ 菜单或长按，降低误触
- "选中态"专门承担批量职责，不与单卡片操作打架
- 后续加新按钮（如"分享"、"打开 N 次记录"）时有明确归属

### 选择框放哪？

你的诉求是"hover 时左上角显示选择框，点击进入选中态"。当前实现是"点击右上角'选择'按钮进入全局选择模式 → 所有卡片渲染换成 `SelectableBookmarkCard`"。两种思路：

**方案 A（推荐）：保留全局选择模式 + 左上角 checkbox 提示**
- 进入选择模式仍走详情页"选择"按钮（成本可见，避免误选）
- 进选择模式后，左上角显示 checkbox（替代当前右上角圆圈，避开 L1 按钮组）
- 卡片右上角 L1 操作按钮在选择模式下隐藏（避免点击歧义）

**方案 B：卡片始终可勾选（hover 即显左上角 checkbox）**
- 任意时刻 hover 都能看到 checkbox
- 一旦勾选某个 → 自动进入选择模式 → 底部工具栏出现
- 全部取消 → 自动退出
- 优点：少一步进入操作；缺点：移动端容易误触，卡片左上角永远占 20px 空间

**我倾向方案 A** —— 进入门槛清晰，避免移动端误触，与现有底部工具栏自然衔接。

### 标签操作怎么放？

- **单卡片改标签** → 走"编辑书签"对话框（已有 TagInput），不需要单独按钮
- **批量加同一个标签** → 多选 → 底部工具栏新增"加标签"按钮 → 弹 TagInput 对话框
- **批量移除某个标签** → 多选 → 底部工具栏新增"移除标签"按钮 → 弹一个"选要移除的标签"对话框（列出选中书签共有的标签让你勾）
- **某个 tag 改名 / 全局删除** → 设置面板的"标签管理"section（未来）

### 收藏夹怎么放？

最轻设计：
- `Bookmark` 加 `isFavorite?: boolean`（store version 2 → 3，迁移：默认 false）
- L1 卡片表面 [★] 切换图标按钮，已收藏时金色填充
- 侧边栏"系统"分组下加一个虚拟项"⭐ 收藏夹"，点击过滤 `isFavorite === true`
- 多选工具栏加"加入收藏夹 / 移出收藏夹"两个按钮

## 关键决策点（你需要拍板）

1. **选择模式入口**：方案 A（保留"选择"按钮进入）还是方案 B（hover 即可勾选）？
2. **删除按钮**：要不要从 L1 完全撤掉，只保留在右键菜单？我建议撤掉，但你可能有不同看法。
3. **L1 三个按钮选谁**：[★ ✎ ⧉] / [★ ✎ 🗑] / 其他组合？
4. **"批量打开 N 个"**：要不要加？打开 20 个 tab 用户可能未必想要，建议加阈值确认（>10 个时弹窗）。
5. **收藏夹是 boolean 还是有序列表**：boolean 简单；有序列表能拖拽排序但工作量翻倍。建议从 boolean 起步。
